sage.ansi¶

Helper methods for ANSI formatting and colors.

Credits for help on this file go to the Lyntin project and Django Termcolors

sage.ansi.colorize(text='', opts=(), **kwargs)[source]¶

Returns your text, enclosed in ANSI graphics codes.

Depends on the keyword arguments ‘fg’ and ‘bg’, and the contents of the opts tuple/list.

Returns the RESET code if no parameters are given.

Parameters:
  • text – string that will be colored
  • opts – (optional) extra options
  • **kwargs – (optional) parameters such as fg and bg
Valid colors:
‘black’, ‘red’, ‘green’, ‘yellow’, ‘blue’, ‘magenta’, ‘cyan’, ‘white’
Valid options:
‘bold’ ‘underscore’ ‘blink’ ‘reverse’ ‘conceal’ ‘noreset’ - string will not be auto-terminated with the RESET code
Examples:
colorize(‘hello’, fg=’red’, bg=’blue’, opts=(‘blink’,)) colorize() colorize(‘goodbye’, opts=(‘underscore’,)) print(colorize(‘first line’, fg=’red’, opts=(‘noreset’,))) print(‘this should be red too’) print(colorize(‘and so should this’)) print(‘this should not be red’)

sage.matching¶

Defines sage’s triggers and aliases.

In sage, a ‘matchable’ is either a trigger or an alias. The behavior of both are identical. The only difference is where they occur in execution.

create(name, mtype, pattern, methods=[], enabled=True, ignorecase=True, delay=None, disable_on_match=False, disable_on_prompt=False, gag=False, intercept=True)[source]¶

Create a trigger or alias (depending on the master group)

Parameters:
  • name (string) – name of the matchable.
  • mtype (string) – ‘type’ of matchable. Must be ‘exact’, ‘substring’, ‘regex’, ‘startswith’, or ‘endswith’.
  • pattern (string) – pattern to match against.
  • methods (list) – (optional) methods to bind to the matchable.
  • enabled (bool) – (optional) is the matchable enabled.
  • ignorecase (bool) – (optional) is the matchable case-insensitive.
  • delay (int or None) – (optional) how many seconds to delay execution of bound methods after a successful match.
  • disable_on_match (bool) – (optional) disable matchable after a successful match.
  • disable_on_prompt (bool) – (optional) disable matchable after a successful match on the following prompt.
  • gag (bool) – (optional) gag the line on a match.
  • intercept (bool) – (optional) -aliases only- Intercept the output of the command being sent.
